HTTP

HTTP

  • Noudattaa asiakas-palvelin-mallia:
    • Asiakas tekee pyynnön → palvelin vastaa
  • Toimii tavallisesti TCP-protokollan päällä
  • 1996: HTTP V1.0  →  1997: HTTP/1.1  →  2015: HTTP/2.0

TCP

TCP

  • Protokolla tiedonsiirtoyhteyden luomiseksi IP-verkossa
  • Luotettava, sisältää virheenkorjausmekanismin
    • Kadonnut paketti lähetetään uudelleen
  • Toimii muun muassa seuraavien protokollien alustana:
    • HTTP
    • SMTP
    • FTP
    • WebSocket
  • 1974: perus-TCP  →  1981: TCP v4

⇑

Alkuperäinen TCP-standardimääritys:

  • RFC675

TCP v4 -standardimääritys:

  • RFC793

WebSocket

WebSocket

  • Protokolla kaksisuuntaiseen datasiirtoon yhden TCP-yhteyden yli
  • Suunniteltu webselain ja -palvelinympäristöön, mutta sovellettavissa myös muuhun asiakas-palvelin-käyttöön
  • Mahdollistaa datan lähettämisen asiakkaalle palvelimen aloitteesta
  • 2011: WebSocket

⇑

WebSocket-standardimääritys:

  • RFC6455
